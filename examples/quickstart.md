# GitConsistency 快速开始示例

本示例展示如何在真实项目中使用 GitConsistency。

## 示例 1: 本地项目分析

```bash
# 1. 安装 GitConsistency
pip install "git-consistency[full]"

# 2. 进入你的项目目录
cd your-project

# 3. 初始化配置
gitconsistency init

# 4. 编辑 .env 文件，添加 API 密钥
# LITELLM_API_KEY=your_deepseek_api_key

# 5. 运行分析
gitconsistency analyze .
```

## 示例 2: GitHub Actions 集成

在项目根目录创建 `.github/workflows/code-review.yml`:

```yaml
name: GitConsistency Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      pull-requests: write
    steps:
      - uses: actions/checkout@v4

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install GitConsistency
        run: pip install "git-consistency[full]"

      - name: Run Review
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          LITELLM_API_KEY: ${{ secrets.LITELLM_API_KEY }}
        run: gitconsistency ci
```

## 示例 3: 仅安全扫描（无需 AI）

```bash
# 只运行安全扫描，不调用 LLM
gitconsistency analyze . --skip-ai

# 输出为 Markdown 文件
gitconsistency analyze . --skip-ai -o security-report.md
```

## 示例 4: 输出格式对比

```bash
# Markdown 格式（默认）
gitconsistency analyze . --format markdown -o report.md

# JSON 格式（用于后续处理）
gitconsistency analyze . --format json -o report.json

# HTML 格式（便于分享）
gitconsistency analyze . --format html -o report.html
```

## 环境变量完整列表

| 变量 | 说明 | 必需 |
|------|------|------|
| `CONSISTENCY_LITELLM_API_KEY` | LLM API 密钥 | AI 审查时必需 |
| `CONSISTENCY_LITELLM_MODEL` | 模型名称，默认 `deepseek/deepseek-chat` | 否 |
| `CONSISTENCY_GITHUB_TOKEN` | GitHub Token，用于 PR 评论 | CI 模式必需 |
| `CONSISTENCY_GITNEXUS_MCP_URL` | GitNexus MCP 地址（实验性） | 否 |

## 配置示例

创建 `.env` 文件：

```bash
# GitConsistency 配置
CONSISTENCY_LITELLM_API_KEY=sk-xxxxxxxxxxxx
CONSISTENCY_LITELLM_MODEL=deepseek/deepseek-chat
CONSISTENCY_GITHUB_TOKEN=ghp_xxxxxxxxxxxx
CONSISTENCY_LOG_LEVEL=INFO
```
